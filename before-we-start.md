Assignment: Designing a Status Server Project

1. Designing a High-Level Architecture
   Create a high-level architecture for your planned status server.

Consider which components are necessary (e.g., frontend, backend, synchronization mechanism, etc.).

Illustrate how these components interact with each other.

Recommended tools: Visio, DrawIO, PowerPoint

2. Defining Communication Protocols and Methods
   Define how the server nodes should synchronize with each other (e.g., REST, WebSocket, RPC, message queue, gossip method, etc.).

Define how communication between the frontend and backend should occur (e.g., HTTP/REST, WebSocket, gRPC).

Justify your selection based on criteria such as performance requirements, simplicity, robustness, or scalability.

Recommended tools: Visio, DrawIO, PowerPoint

4. Implementation of a Proof of Concept (PoC)

Develop a simple demonstration of your status server solution. (A full implementation of the status server is not necessary.)

Show how the chosen communication methods were implemented.

5. Short presentation (max. 5 minutes)

Architecture: Present a block diagram of the components and their interaction.

Communication methods: Explain the protocols and procedures used, including justification.

Proof of Concept: Brief demonstration of the implemented solution.
